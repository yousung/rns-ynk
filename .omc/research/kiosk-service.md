## 1. 키오스크 서비스의 목적과 주요 기능

키오스크 페이지(`/kiosk`)는 창고 현장에 설치된 **고정형 디스플레이(키오스크 단말기)** 를 위한 상태 모니터링 전용 화면이다.

사용자가 직접 조작하는 UI가 아니라, **시스템 연결 상태와 활동 로그를 실시간으로 표시**하는 대시보드형 화면이다.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 실시간 시계 | 현재 시각(HH:MM:SS) 및 날짜(요일 포함) 표시 |
| 블루투스 연결 상태 표시 | 전동랙 BT-001의 연결/재시도/실패 상태를 시각적으로 표시 |
| 시스템 활동 로그 | success/info/warn/error 타입으로 분류된 실시간 로그 스트림 (최대 50건) |
| 모바일 연동 QR 코드 | 장식용 SVG QR 코드 — 모바일 앱 연동 안내 |
| 서버 IP 표시 | 내부망 서버 주소(192.168.1.100:8080) 상시 표시 |
| 라이트/다크 테마 토글 | 화면 우측 상단 버튼으로 전환 가능 |

## 2. UI 흐름 (사용자가 어떻게 사용하는지)

키오스크는 **자동 실행(상시 표시)** 화면으로, 사용자의 직접 조작을 거의 요구하지 않는다.

```
[시스템 기동]
    ↓
/kiosk 페이지 자동 로드
    ↓
초기 로그 10건 표시 (INIT_LOGS)
+ BT 연결 상태 "연결됨" 표시
+ 시계 시작 (1초 인터벌)
    ↓
[자동 갱신 루프]
    - 매 3~5초: 랜덤 로그 추가 (RANDOM_LOGS, 70%) 또는 BT 상태 로그 (30%)
    - 매 6초: BT 상태 순환 (연결됨 → 재시도 중 → 연결 실패 → ...)
    - 로그는 최신순, 최대 50건 유지
    ↓
[운영자 선택적 조작]
    - 다크/라이트 모드 토글 (1개 버튼)
```

## 3. 데이터 연동 방식

### Store 연동
- **useDataStore 미사용**: 창고·재고·팔레트 등 비즈니스 데이터와 완전히 분리됨
- **useUIStore 미사용**: 전역 테마 store를 사용하지 않고, 로컬 `useState`로 독립 테마 관리
- **useAuthStore 미사용**: 인증 없이 접근 가능 (`RequireAuth` 래퍼 없음)

### 데이터 소스
- 모든 로그/상태 데이터는 **컴포넌트 내부 상수(`BT_STATES`, `INIT_LOGS`, `RANDOM_LOGS`)**로 하드코딩
- 외부 API 호출 없음 — 완전한 데모/시뮬레이션 데이터

### 상태 관리 (로컬)
| 상태 | 타입 | 역할 |
|------|------|------|
| `theme` | useState | 라이트/다크 모드 |
| `now` | useState | 현재 시각 (1초 갱신) |
| `btIdx` | useState | BT 상태 인덱스 (6초 순환) |
| `logs` | useState | 활동 로그 배열 (최대 50건) |
| `logIdRef` | useRef | 로그 고유 ID 카운터 |
| `btTimeoutRef` | useRef | 랜덤 로그 setTimeout 핸들 |

## 4. 웹 서비스(`/app`)와의 차이점

| 항목 | 웹 서비스 (`/app`) | 키오스크 (`/kiosk`) |
|------|-------------------|-------------------|
| 인증 | 필요 (RequireAuth) | 불필요 |
| 레이아웃 | 사이드바 + 헤더 포함 (Layout 컴포넌트) | 전용 풀스크린 레이아웃 |
| 전역 Store 사용 | useDataStore, useUIStore, useAuthStore | 미사용 |
| 사용자 조작 | CRUD 중심 (입고/출고/재고 관리) | 최소화 (테마 토글만) |
| 데이터 | 실제 비즈니스 데이터 연동 | 하드코딩 시뮬레이션 |
| 화면 수 | 다수 페이지 (라우팅) | 단일 페이지 |
| CSS | 전역 + 컴포넌트 스타일 | `kiosk.css` 전용 스타일 |

## 5. 키오스크 특화 기능

### 터치 UI / 단순화된 인터페이스
- **조작 요소 최소화**: 테마 토글 버튼 1개만 노출
- **대형 텍스트**: 시계(`k-clock-time`)는 큰 폰트로 원거리에서도 가독성 확보 (CSS 추정)
- **자동 갱신**: 사용자 조작 없이 로그/상태가 자동 업데이트 → 상시 무인 운영 가능

### 현장 특화 정보
- **블루투스 연결 상태**: 전동랙 BT-001 장치 상태를 3단계(연결됨/재시도/실패)로 시각적 표시
  - `bt-connected`: 파란색 펄스 애니메이션 (anim-pulse)
  - `bt-retrying`: 노란색 점멸 애니메이션 (anim-blink)
  - `bt-failed`: 빨간색 정적 아이콘
- **내부망 전용**: 푸터에 "내부망 전용" 명시, 서버 IP 192.168.1.x 대역

### 운영 정보 표시
- 버전 정보 (`v1.0.0-demo`) 및 빌드 날짜 푸터 표시
- WMS 브랜딩 헤더 + "KIOSK" 배지로 화면 유형 명확히 구분

## 요약

키오스크 서비스는 창고 현장 설치용 **상태 모니터링 전용 화면**이다. 웹 서비스와 완전히 독립된 아키텍처(인증 없음, 전역 store 미사용, 전용 CSS)를 갖추며, 전동랙 블루투스 연결 상태와 시스템 로그를 실시간으로 시뮬레이션하여 표시한다. 현재는 모든 데이터가 하드코딩된 데모 상태이며, 실제 배포 시에는 BT 장치 API 및 서버 로그 스트림 연동이 필요하다.
