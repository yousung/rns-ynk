## 1. 태블릿 서비스의 목적과 주요 기능

태블릿 서비스는 **창고 현장 작업자가 태블릿/모바일 기기로 입고·출고 작업을 직접 처리**하기 위한 인터페이스다. 관리자용 웹 대시보드(`/app`)와 달리 복잡한 메뉴 구조가 없고, 현장에서 한 손으로 조작 가능한 터치 최적화 UI로 구성된다.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 입고 처리 | 입고 예정 목록에서 상품 선택 → 창고/랙/층/칸 선택 → 수량 입력 → 확정 |
| 출고 처리 | 출고 예정 목록에서 상품 선택 → FIFO 순 재고 로트 선택 → 수량 입력 → 확정 |
| 전동랙 제어 | 전동랙 선택 시 열기/멈춤 버튼 노출 (시그널 전송 UI) |
| 다크/라이트 테마 전환 | 좌상단 버튼으로 현장 조명 환경에 따라 전환 |
| 화면 간 이동 | 하단 탭바로 입고처리 ↔ 출고처리 ↔ 로그아웃 전환 |

## 2. 입고(Inbound)와 출고(Outbound) 화면의 차이

### TabletInbound (입고 처리)
- **창고 유형 토글**: 일반 창고 / 전동랙 전환 (상단)
- **위치 지정 흐름**: 창고 선택 → 랙 선택 → 층 선택 → 칸 선택 (단계별 순차 표시)
- **점유 칸 비활성화**: `isOccupied()` 함수로 이미 팔레트가 있는 칸은 `occupied` 스타일로 disable 처리
- **전동랙 제어**: 전동랙 모드 + 랙 선택 시 열기/멈춤 버튼 표시 (토스트 메시지로 피드백)

### TabletOutbound (출고 처리)
- **FIFO 강제**: 재고 로트를 `received_at` 오름차순으로 정렬, 순번(#1, #2...) 표시
- **로트 카드 선택**: inventoryItems → pallet → 위치 역참조로 창고/랙/층/칸 정보 표시
- **출고 위치 대형 표시**: 선택한 로트의 위치를 큰 글씨(2rem, cyan glow)로 강조 — 현장 작업자가 즉시 확인 가능
- **전동랙 상태 표시**: 전동랙 창고 로트 선택 시 `idle → running → stopped` 상태 UI 제공
- **FIFO 배지**: 패널 헤더에 FIFO 배지 표시로 정책 명시

### 핵심 차이 요약
| 항목 | 입고 | 출고 |
|------|------|------|
| 주요 선택 흐름 | 위치를 작업자가 직접 선택 | 시스템이 FIFO 순서로 위치 제안 |
| 전동랙 제어 | 열기/멈춤 토스트 피드백 | 열기/멈춤 + 상태 표시줄 |
| 위치 표시 방식 | 선택 배너에 소형 텍스트 | 별도 블록에 대형 숫자 표시 |
| 창고 유형 전환 | 상단 토글 버튼 | 자동(로트가 속한 창고에 따라) |

## 3. UI 흐름 (현장 작업자 사용 시나리오)

### 입고 처리 흐름
```
1. 왼쪽 패널: 입고 예정 카드 클릭 (상품명, 수량, 날짜, 메모 확인)
2. 우측 상단 토글: 일반 창고 또는 전동랙 선택
3. 창고 선택 버튼 클릭 (일반 창고만 해당)
4. 랙 선택 → 층 선택 → 칸 선택 (단계별 그리드 버튼)
5. 점유된 칸은 amber 색상으로 비활성화
6. 전동랙이면: 열기 버튼으로 랙 이동 신호 전송
7. 하단 확정 바: 수량 입력 → 입고 확정 버튼 클릭
8. 토스트 메시지로 완료 확인
```

### 출고 처리 흐름
```
1. 왼쪽 패널: 출고 예정 카드 클릭 (상품명, SKU, 수량, 납품처 확인)
2. 우측: FIFO 순 재고 로트 목록 표시 (입고일, 위치, 재고량)
3. 로트 선택 → 출고 위치(창고/랙/층/칸)가 대형 텍스트로 표시
4. 전동랙 창고이면: 열기/멈춤 버튼으로 랙 제어 + 상태 표시
5. 수량 입력 → 출고 확정
6. 토스트 메시지로 완료 확인 (상품명, 수량, 납품처 포함)
```

## 4. 데이터 연동 방식

### Store 구조
- **`useDataStore`** (Zustand): 정적 더미 데이터 기반. 창고/랙/팔레트/상품/재고/입출고예정 데이터를 메모리에서 직접 읽음
- **`useAuthStore`**: 태블릿 페이지에서는 **사용하지 않음** — `/tablet` 라우트는 `RequireAuth` 보호 없이 직접 접근 가능

### 데이터 흐름
```
useDataStore
  ├── warehouses     → 창고 목록 (normal/electric 타입 분류)
  ├── racks          → 창고별 랙 (floors, groups로 층/칸 수 정의)
  ├── pallets        → 팔레트 위치 정보 ("rackId-floor-kan" 포맷)
  ├── products       → 상품 정보 (코드, 이름, 카테고리)
  ├── inventoryItems → 재고 (product_id, pallet_id, quantity, received_at)
  ├── inboundSchedules  → 입고 예정 (pending/done 상태)
  └── outboundSchedules → 출고 예정 (pending/done 상태)
```

### 위치 인코딩 방식
- 팔레트 위치: `"${rackId}-${floor}-${kan}"` 문자열 포맷
- 출고 시 역참조: `pallet.location.split('-').map(Number)` → `[rackId, floor, kan]`

### 현재 상태
- API 연동 없음 (순수 클라이언트 상태)
- 입고/출고 확정 시 실제 store 업데이트 없음 (토스트 표시 후 선택 초기화만 수행)
- **실제 서비스 전환 시 API 연동 및 store mutation 구현 필요**

## 5. 웹 서비스 / 키오스크와의 차이점

| 항목 | 웹 서비스 (`/app`) | 태블릿 (`/tablet`) | 키오스크 (`/kiosk`) |
|------|-------------------|--------------------|---------------------|
| 대상 사용자 | 관리자, 사무직 | 창고 현장 작업자 | 방문자/직원 셀프 서비스 |
| 인증 | RequireAuth (필수) | 없음 (자유 접근) | 별도 구조 |
| 레이아웃 | 사이드바 + 헤더 풀레이아웃 | 헤더 + 탭바 2컬럼 | 별도 |
| 화면 수 | 9개 이상 (대시보드, 재고, 설정 등) | 2개 (입고/출고) | 별도 |
| 기능 범위 | 전체 WMS 관리 | 입고/출고 처리만 | 별도 |
| UI 밀도 | 정보 밀도 높음 (테이블, 차트) | 터치 타겟 크고 단순 | 별도 |
| 창고 시각화 | 매트릭스, 미니맵, 플로어플랜 | 없음 (텍스트/버튼만) | 별도 |

## 6. 태블릿 특화 기능

### 터치 최적화 UI
- 모든 버튼 최소 높이 `52px` (`.t-grid-btn`), 확정 버튼 `50px` — 손가락 터치 오류 최소화
- `-webkit-tap-highlight-color: transparent` — 모바일 탭 하이라이트 제거
- `word-break: keep-all` — 한국어 줄바꿈 최적화

### 다크/라이트 테마
- 기본: 다크 모드 (`--bg-base: #070C14`) — 창고 조명 환경에서 눈부심 감소
- CSS 변수 기반으로 `data-theme="light"` 속성 전환
- 고정 위치(`position: fixed`) 전환 버튼 — 항상 접근 가능

### 점유 셀 실시간 표시
- 입고 시 `isOccupied()` 함수로 팔레트 데이터와 대조 — 이미 사용 중인 칸 시각적 구분

### FIFO 강제 적용
- 출고 로트를 `received_at` 기준 정렬하여 오래된 재고 우선 출고 유도
- 순번 배지(#1, #2) + amber 색상으로 FIFO 우선순위 직관적 표시

### 전동랙 제어 통합
- 입고: 전동랙 선택 시 열기/멈춤 버튼 즉시 노출
- 출고: 전동랙 창고 로트 선택 시 상태 표시줄(`idle → running → stopped`) + 제어 버튼
- 물리적 장비 제어와 WMS 작업을 단일 화면에서 처리

### 간결한 네비게이션
- 하단 탭바 3개(입고처리 / 출고처리 / 로그아웃) — 현장에서 빠른 화면 전환
- 서브메뉴 없음, 중간 단계 화면 없음

## 7. 구조적 특이사항

1. **인증 없는 직접 접근**: `/tablet/*` 라우트는 `RequireAuth` 래퍼 없이 등록되어 있어 로그인 없이 접근 가능. 현장 운영상 편의를 위한 설계로 보이나, 실서비스 전 보안 정책 결정 필요.

2. **단방향 데이터**: 확정 버튼 클릭 시 store를 실제로 변경하지 않아 새로고침 시 원복됨. 프로토타입 단계.

3. **창고 유형 자동 분기**: 전동랙(`type: 'electric'`) 여부를 warehouses 데이터의 type 필드로 판단. 출고에서는 선택된 로트의 창고 타입을 `isElectric` 변수로 자동 감지.
