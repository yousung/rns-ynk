# 태블릿/키오스크 분석

## 태블릿 — 입고 처리 흐름 (TabletInbound.jsx)

### 주요 단계
1. **입고 예정 선택** (좌측 패널)
   - 펜딩 상태의 입고 예정 목록 표시
   - 상품명, 수량, 예정일 표시
   - 클릭으로 선택

2. **창고 유형 선택** (우측 상단)
   - 토글 버튼: "일반 창고" ↔ "전동랙"
   - 모드 변경 시 선택 상태 초기화

3. **위치 선택 (계층적 드릴다운)**
   - 창고 선택 → 랙 선택 → 칸 선택 → 단 선택
   - 각 단계마다 이전 선택 초기화
   - 단 선택 시 점유 여부 확인 (occupied 상태)

4. **전동랙 제어** (전동랙 모드만)
   - "열기" 버튼: ▶ 신호 전송
   - "멈춤" 버튼: ■ 신호 전송
   - 토스트 메시지로 피드백

5. **수량 입력 및 확정**
   - 입력된 수량 유효성: 1 이상, 예정 수량 이하
   - "입고 확정" 버튼: 조건 만족 시만 활성화
   - 완료 후 선택 상태 초기화

### 현장 조작 포인트
- **터치 UI 특징**: 버튼이 크고 명확 (t-grid-btn), 단계별 진행
- **시각적 피드백**: 선택된 항목 강조 (active 클래스), 토스트 메시지
- **입력 제약**: 점유된 위치 비활성화 (disabled), 수량 범위 체크
- **모드 전환**: 창고 유형 변경 시 선택 초기화로 혼동 방지

## 태블릿 — 출고 처리 흐름 (TabletOutbound.jsx)

### 주요 단계
1. **출고 예정 선택** (좌측 패널)
   - 펜딩 상태의 출고 예정 목록 표시
   - **FIFO 배지** 강조 (FIFO 원칙 시각화)
   - 상품명, SKU 코드, 수량, 예정일, 목적지(note) 표시

2. **FIFO 로트 선택** (우측 상단)
   - `getProductLots()`: 선택 상품의 재고를 입고일 순서로 정렬
   - 각 로트별 로케이션 정보 표시
   - "#1, #2..." 순번으로 FIFO 순서 가시화
   - 로트 선택 시 칸/단 자동 로드

3. **출고 위치 확인** (우측 중단)
   - "출고 위치" 섹션: 창고/랙/칸/단 계층적 표시
   - 입고일(received_at) 함께 표시
   - 수동 선택 불필요 (FIFO 선택으로 자동 결정)

4. **전동랙 제어** (전동랙 모드만)
   - 상태 표시: "대기 중" → "이동 중" → "정지됨"
   - 상태별 메시지: `elecStatusText` 동적 표시
   - "열기" → "멈춤" 시퀀스로 운영

5. **수량 입력 및 확정**
   - 선택된 로트 재고 수량 이내 입력
   - "출고 확정" 버튼: 전체 조건 만족 시만 활성화
   - 완료 후 선택 초기화

### 현장 조작 포인트
- **FIFO 강제화**: 로트 목록이 입고일순 정렬되어 순서 실수 방지
- **자동화**: 로트 선택 시 위치 자동 결정 (수동 선택 없음)
- **시각화**: 로트별 순번(#1, #2...) + 입고일 표시로 FIFO 원칙 강조
- **상태 추적**: 전동랙 상태를 텍스트로 명확히 안내

## 키오스크 — 기능 목록 (KioskPage.jsx)

### 시스템 상태 모니터링
1. **실시간 시계**
   - `fmtTime()`: HH:MM:SS 형식
   - `fmtDate()`: YYYY.MM.DD (요일) 형식
   - 1초 간격 업데이트 (useEffect + setInterval)

2. **블루투스 (BT) 연결 상태**
   - 상태 3가지: connected(🔵) / retrying(🟡) / failed(🔴)
   - 신호 강도 표시 (-65dBm 등)
   - 상태별 애니메이션: anim-pulse / anim-blink
   - 6초마다 자동 상태 전환 (BT_STATES 순환)

3. **QR 코드 (모바일 연동)**
   - 고정 SVG QR 코드 패턴 (장식용)
   - "스캔하여 모바일 앱 연동" 안내

4. **서버 주소 표시**
   - 192.168.1.100:8080 (내부망 전용)

5. **시스템 활동 로그**
   - 초기화 로그: 10개 (시작 시퀀스)
   - 런타임 로그: 3~5초 간격으로 추가
   - 로그 유형: success(✓) / info(ℹ) / warn(⚠) / error(✗)
   - 최대 50개 항목 유지 (slice(0, 50))
   - BT 상태와 연계: 30% 확률로 BT 로그, 70% 확률로 랜덤 로그

### 설정 및 테마
- 테마 토글: Light ↔ Dark (styles/kiosk.css 참고)
- 앱 업데이트 버튼: 확인 중 토스트 표시
- 모달 방식으로 구현

### 풋터 및 버전
- 빌드 날짜: 2026-04-20
- 명칭: "내부망 전용"
- 버전: v1.0.0 (우측 하단 고정)

### 현장 배치 포인트
- **수동 조작 최소화**: 자동 시계, 자동 로그 업데이트
- **상태 모니터링**: BT 연결, 서버 연결 상태 한눈에
- **운영자 모드**: 로그 확인으로 시스템 정상 여부 판단
- **터치 UI**: 대형 버튼, 명확한 색상 코드 (🔵/🟡/🔴)

## 현장 작업자 관점 포인트

### 입고 작업자
1. **명확한 단계별 진행**: 창고 → 랙 → 칸 → 단 (드릴다운)
2. **시각적 제약**: 점유된 위치는 클릭 불가(disabled) → 실수 방지
3. **이중 확인**: 위치 선택 후 "입고 확정" 버튼으로 최종 확인
4. **토스트 피드백**: 완료 메시지로 즉각적 반응

### 출고 작업자
1. **FIFO 자동화**: 로트 목록이 입고일순 → 수동 확인 불필요
2. **자동 위치 결정**: 로트 클릭 시 위치 자동 로드 → 빠른 처리
3. **순서 가시화**: 로트 순번(#1, #2...) 표시로 FIFO 원칙 강조
4. **상태 안내**: 전동랙 상태를 텍스트로 명확히 표시

### 공통 UX 패턴
- **설정 모달**: 프로필 편집, 비밀번호 변경, 테마 선택
- **탭 네비게이션**: 입고처리 ↔ 출고처리 ↔ 설정 (t-tab-bar)
- **로그아웃 버튼**: 설정 모달 하단에 배치
- **라이트/다크 테마**: 시간대별 환경에 대응

### 키오스크 운영 관점
1. **자동 모니터링**: 운영자는 화면만 보고 시스템 상태 판단
2. **로그 추적**: BT 신호 강도, 서버 응답, 동기화 상태 실시간 확인
3. **장비 감지**: BT 연결 실패 시 즉시 시각화(🔴) → 빠른 대응
4. **모바일 연동**: QR 코드로 모바일 앱 연결 (터치리스 입력)
